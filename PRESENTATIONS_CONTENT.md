## Common Animation Patterns Across Presentations

### Fade-In Animation
- Most text and elements use alpha blending
- Typical pattern: `alpha = min(1.0, (progress - delay) / duration)`
- Smooth transitions from 0 to 1 opacity

### Sequential Reveals
- Elements appear one after another
- Each element has a delay offset
- Creates flowing, storytelling effect

### Bounce/Scale Effects
- Icons and cards often scale in
- Formula: scale from 0.5 to 1.0 with ease-out
- Adds energy and draws attention

### Progress Indicators
- Animated dots or spinner during transitions
- Shows animation is in progress
- Typically in accent color

### Color Coding
- Consistent color scheme across all presentations:
  - Primary: Blue tones (main concepts)
  - Secondary: Green tones (positive/success)
  - Accent: Yellow/orange (highlights)
  - Warning: Red tones (caution/problems)
  - Purple: Advanced concepts
  - Cyan: Technical details

### Box Layouts
- FancyBboxPatch with rounded corners
- Colored borders matching content theme
- Semi-transparent backgrounds
- Shadow effects for depth

### Flow Animations
- Arrows drawing between connected elements
- Data "flowing" through pipelines
- Pulse effects to show signal transmission
- Typically uses FancyArrowPatch

### Text Effects
- Typewriter effect for code/examples
- Emphasis through font weight and color
- Italic for quotes or secondary info
- Monospace for technical content
